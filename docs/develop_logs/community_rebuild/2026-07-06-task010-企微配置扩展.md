## 完成内容

### 后端 API

| Method | Path | 说明 |
|--------|------|------|
| GET | `/org-wecom/config/get?organizationId=` | 读取组织同步配置（Secret 掩码） |
| POST | `/org-wecom/config/test` | 测试 gettoken + listDepartments |
| POST | `/org-wecom/config/save` | 已有，增强掩码占位保存逻辑 + 审计日志 |

**新增类：**

- `OrgWecomSyncConfigDTO` — GET 响应
- `OrgWecomSyncConfigTestRequest` / `OrgWecomSyncConfigTestResponse`
- `OrgWecomSyncSecretUtils` — `******` + 后 4 位掩码

**安全：**

- GET 返回 `******` + Secret 后 4 位
- 保存时若 `contactSecret` 以 `******` 开头则跳过更新
- save / test 写 `OperationLogService` 审计日志

### 前端

**方案 A**：在 `orgStructure/index.vue` 增加「同步配置」按钮，打开 `SyncConfigDrawer.vue`

**配置项：**

| 字段 | 组件 |
|------|------|
| corpId | Input |
| contactSecret | Input.Password（掩码回显） |
| scheduleEnabled | Switch |
| scheduleCron | MsCronSelect + Cron 帮助链接 |
| retryTimes | InputNumber（默认 3） |

**提示文案**：明确区分通讯录 Secret 与登录 Secret

## 验证建议

1. 组织架构页点击「同步配置」，首次保存 corpId + contactSecret + cron
2. 再次打开配置，Secret 显示掩码；不修改 Secret 直接保存，原值不变
3. 点击「测试连接」，成功 toast 显示 deptCount；错误 Secret 显示 errcode
4. 保存后执行「同步企微」，确认手动同步正常
