## 1. 本次目标

暴露企微组织架构同步 HTTP API，并实现基于 Quartz 的定时同步任务。

---

## 2. API 清单

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/org-wecom/sync/manual` | 手动触发同步，返回统计 |
| GET | `/org-wecom/sync/status` | 最近同步状态 + `last_sync_time` |
| GET | `/org-wecom/sync/log/page` | 同步日志分页 |
| POST | `/org-wecom/config/save` | 保存企微同步配置 |

**权限**：系统管理员 `SYSTEM_ORGANIZATION_PROJECT:READ+UPDATE` 或组织管理员 `ORGANIZATION_MEMBER:READ+UPDATE`（读操作为 READ）。

---

## 3. 核心实现

| 组件 | 路径 |
|------|------|
| `OrgWecomSyncController` | 四个 API 入口 |
| `OrgWecomSyncQueryService` | 状态查询、日志分页 |
| `OrgWecomSyncConfigService` | 配置保存 + 触发 Quartz 刷新 |
| `WecomOrgSyncScheduleService` | 启动时初始化 / 保存时动态注册 Cron |
| `WecomOrgSyncJob` | Quartz Job，调用 `syncSchedule()` |
| `ExtOrgSyncLogMapper` | 最近日志、分页列表 |

**并发控制**：同步进行中再次请求返回 HTTP **409**（`MsHttpResultCode.CONFLICT`），Redis 锁 TTL 30 分钟。
