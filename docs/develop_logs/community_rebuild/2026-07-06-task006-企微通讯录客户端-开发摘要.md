## 1. 本次目标

新建独立于 Webhook 的企微通讯录 API 客户端，封装 Token 管理、部门/成员拉取能力，供 task007 同步引擎调用。

---

## 2. 核心实现

| 组件 | 路径 | 说明 |
|------|------|------|
| `WecomContactClient` | `service/wecom/WecomContactClient.java` | gettoken 内存缓存（过期前 5 分钟刷新）、department/list、user/list、user/get |
| `WecomContactConfigProvider` | `service/wecom/WecomContactConfigProvider.java` | 按 `organizationId` 读取 `org_wecom_sync_config` |
| `WecomApiException` | `service/wecom/WecomApiException.java` | errcode != 0 统一异常；42001/40014 识别为 Token 过期 |
| DTO | `dto/wecom/*` | `WecomDepartmentDTO`、`WecomUserDTO`、`WecomTokenResponse` 等 |

**与现有代码边界**：未改动 `notice/utils/WeComClient`（Webhook）及 `WeComNoticeSender`。

---

## 3. 能力说明

- **Token 缓存**：`corpId + contactSecret` 为 key，双重检查锁刷新
- **Token 过期重试**：`executeWithToken(corpId, secret, action)` 遇 42001 清缓存并重试 1 次
- **网络重试**：默认 3 次指数退避（`wecom.contact.retry-times` 可配置）
- **测试 API 地址**：`wecom.api-base-url`（默认 `https://qyapi.weixin.qq.com`）
