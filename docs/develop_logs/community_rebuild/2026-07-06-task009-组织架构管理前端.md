# 2026-07-06 task009 组织架构管理前端

## 完成内容

### 路由与菜单

- 系统设置 `/setting/system/org-structure`（`SYSTEM_ORGANIZATION_PROJECT:READ`）
- 组织设置 `/setting/organization/org-structure`（`ORGANIZATION_MEMBER:READ`）
- 菜单 i18n：`menu.settings.system.orgStructure` / `menu.settings.organization.orgStructure`

### 页面与组件

```
frontend/src/views/setting/system/orgStructure/
├── index.vue                    # 左树右表 + 组织选择 + 同步操作
├── components/
│   ├── DepartmentTree.vue       # 部门树（含人数、停用灰显）
│   ├── MemberTable.vue          # 成员分页表 + 筛选
│   ├── MemberDetailDrawer.vue   # 脱敏详情抽屉
│   └── SyncPanel.vue            # 手动同步 + 状态 + 日志抽屉
├── config.ts
└── locale/zh-CN.ts, en-US.ts
```

### API 封装

- `frontend/src/api/requrls/setting/orgStructure.ts`
- `frontend/src/api/modules/setting/orgStructure.ts`
- `frontend/src/models/setting/orgStructure.ts`

对接后端：

| API | 用途 |
|-----|------|
| GET `/org-structure/departments/tree` | 部门树 |
| GET `/org-structure/members/page` | 成员分页 |
| GET `/org-structure/members/{id}` | 成员详情 |
| POST `/org-wecom/sync/manual` | 手动同步 |
| GET `/org-wecom/sync/status` | 最近同步状态 |
| GET `/org-wecom/sync/log/page` | 同步日志 |

### 交互要点

- 系统管理员可通过组织下拉查看任意组织数据
- 组织管理员固定当前组织
- 手动同步完成后自动刷新树、表、状态
- 同步冲突（409）提示友好文案
- 成员 syncStatus：0 未同步 / 1 已同步 / 2 失败
