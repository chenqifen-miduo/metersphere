## 1. 本次目标

实现企微通讯录 → MS 组织内部门/用户的幂等全量同步引擎，含部门两阶段 upsert、员工 upsert、失活收敛与安全保护。

---

## 2. 模块结构

| 类 | 职责 |
|----|------|
| `WecomOrgSyncApplicationService` | 编排、Redis 组织级锁、`org_sync_log` 写入、`last_sync_time` 更新 |
| `WecomOrgSyncService` | `syncAll()`：先部门后员工，聚合 `SyncResult` |
| `DepartmentSyncHandler` | Pass1 upsert、Pass2 parent_id、Pass3 失活 |
| `UserSyncHandler` | 根部门拉成员、创建/更新用户、`org_member` 绑定、失活 |
| `SyncResult` | 同步统计 DTO |

---

## 3. 安全保护

- 企微返回空部门/空成员列表 → **跳过失活**
- `mobile`/`position` 空值不覆盖本地已有值
- `admin` 用户与 `DEV_` 前缀企微账号不参与失活
- 单组织 Redis 锁 `ms:wecom:org-sync:{organizationId}`
