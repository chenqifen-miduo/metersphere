# BUG001：创建用户成功但列表不显示

> **日期**：2026-06-26  
> **关联任务**：task001（社区版 Xpack / License 实现）  
> **影响范围**：系统设置 → 用户管理（手动创建、Excel 导入、邀请注册、启用/禁用、删除）  
> **严重程度**：高（核心用户管理不可用）

---

## 1. 问题现象

### 1.1 阶段一：接口 500 / NPE

在系统设置中手动创建用户时，前端报错；后端日志类似：

```
NullPointerException
at SimpleUserService.saveUserAndRole(...:135)
at Objects.requireNonNull(CommonBeanFactory.getBean(UserXpackService.class)).GWHowToAddUser(...)
```

同时 `GET /license/validate` 返回 `{ "status": null }`，说明 `LicenseService` Bean 也未注册。

### 1.2 阶段二：提示成功但列表无数据

修复 Bean 注册并完成 task001 初版后：

- 前端提示「创建成功」
- 用户列表刷新后**看不到新用户**
- 接口 `POST /system/user/add` 返回 `code=100200`（业务成功）

---

## 2. 根因分析

MeterSphere V3 开源代码中，**用户 CRUD 的持久化逻辑不在 `SimpleUserService` 内**，而是通过企业版扩展点 `UserXpackService` 完成。社区版源码未提供企业 `general-xpack-impl` JAR，需自行实现 Community 版本。

### 2.1 Bean 未注册（阶段一）

task001 初版在实现类上使用 `@Service` + `@ConditionalOnMissingBean`：

```java
@Service
@ConditionalOnMissingBean(UserXpackService.class)
public class CommunityUserXpackServiceImpl implements UserXpackService { ... }
```

在 Spring Boot 3.5 运行时，上述方式**未能可靠注册 Bean**，导致：

- `CommonBeanFactory.getBean(UserXpackService.class)` 返回 `null`
- `saveUserAndRole()` 第 135 行触发 NPE

### 2.2 仅返回成功码、未写库（阶段二）

`SimpleUserService.saveUserAndRole()` 在 xpack 返回 `0` 后**只写操作日志**，不会执行 `userMapper.insert`：

```134:145:backend/services/system-setting/src/main/java/io/metersphere/system/service/SimpleUserService.java
    private List<UserCreateInfo> saveUserAndRole(UserBatchCreateRequest userCreateDTO, String source, String operator, String requestPath) {
        int responseCode = Objects.requireNonNull(CommonBeanFactory.getBean(UserXpackService.class)).GWHowToAddUser(userCreateDTO, source, operator);
        if (responseCode == 0) {
            operationLogService.batchAdd(userLogService.getBatchAddLogs(userCreateDTO.getUserInfoList(), operator, requestPath));
        } else {
            // ...
        }
        return userCreateDTO.getUserInfoList();
    }
```

task001 初版 `CommunityUserXpackServiceImpl` 四个方法均 `return 0`，等价于「配额检查通过」，但**没有插入 user 表、没有写 user_role_relation**。  
企业版 `GWHowToAddUser` 负责实际落库；Community 桩实现遗漏该职责。

### 2.3 前端并非问题

用户管理页在创建成功后会调用 `loadList()` 刷新列表，刷新逻辑正常。  
列表数据来自 `baseUserMapper.selectByKeyword(..., false)`，条件为 `deleted = 0`；库中无记录则列表为空。

### 2.4 cftToken 要求（登录相关）

`user` 表字段 `cft_token` 用于 monitoring-engine 反篡改校验。若直接 SQL 插入或使用 `"NONE"`，可能出现「检测到用户创建途径异常，禁止登录」。  
正确做法：创建用户时设置 `cftToken = CFTEncryptUtils.aesEncrypt(userId)`。

---

## 3. 修复方案

### 3.1 统一 Bean 注册（解决阶段一）

新增 `CommunityXpackConfiguration`，采用与 `I18nConfig` 相同的 `@Configuration` + `@Bean` + `@ConditionalOnMissingBean` 模式：

| Bean | 实现类 |
|------|--------|
| `LicenseService` | `CommunityLicenseServiceImpl` |
| `UserXpackService` | `CommunityUserXpackServiceImpl` |

实现类改为普通 POJO，由配置类注入依赖并注册。

**文件**：`backend/services/system-setting/src/main/java/io/metersphere/system/config/CommunityXpackConfiguration.java`

### 3.2 Community 版用户持久化（解决阶段二）

在 `CommunityUserXpackServiceImpl` 中补全与企业版等价的落库逻辑：

| 方法 | 修复后行为 |
|------|-----------|
| `GWHowToAddUser(batch)` | 循环插入 `user`；默认密码 `md5(email)`；写 `cft_token`；调用 `updateUserSystemGlobalRole` 分配系统用户组 |
| `GWHowToAddUser(register, invite)` | 插入用户；密码 `md5(registerRequest.password)`；按邀请范围（系统/组织/项目）写 `user_role_relation` |
| `GWHowToChangeUser` | 更新 `user.enable` |
| `GWHowToDeleteUser` | 软删除 `user.deleted = true`，返回成功条数 |

**文件**：`backend/services/system-setting/src/main/java/io/metersphere/system/service/impl/CommunityUserXpackServiceImpl.java`

### 3.3 单元测试调整

`CommunityUserXpackServiceImplTest` 改为 Mockito 注入 `UserMapper` / `UserRoleRelationMapper` / `UserRoleRelationService`，避免无依赖构造导致 NPE；删除用户用例断言改为返回 `1`（实际删除条数）。

**文件**：`backend/services/system-setting/src/test/java/io/metersphere/system/service/impl/CommunityUserXpackServiceImplTest.java`

## 6. 经验与后续

1. **task001 文档需同步**：初版文档写「返回 0 后继续 insert user」与 `SimpleUserService` 实际代码不符；Community 实现必须自行完成 insert。
2. **Bean 注册方式**：社区版扩展点优先使用 `@Configuration` + `@Bean`，不要仅依赖实现类上的 `@Service` + `@ConditionalOnMissingBean`。
3. **与企业版关系**：若引入 `general-xpack-impl` JAR，`@ConditionalOnMissingBean` 会跳过 Community 实现，行为以企业版为准。
4. **关联文档**：[task001 代码详细说明](../community_rebuild/details/task001-社区版Xpack与License-代码详细说明.md)
