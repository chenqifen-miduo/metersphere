# BUG-PRJ-001：编辑项目「所属组织」更新不生效

> **日期**：2026-07-04  
> **关联任务**：task002 / task003 联调  
> **影响范围**：系统设置 → 组织与项目 → 编辑项目  
> **严重程度**：P1（功能）

---

## 1. 问题现象

在系统设置中修改项目的「所属组织」并保存，列表或再次打开编辑框仍显示原组织。

---

## 2. 根因分析

`CommonProjectService.update()` 在调用 `updateByPrimaryKeySelective` 前执行：

```java
project.setOrganizationId(null);
```

导致 selective update **永不写入**新的 `organization_id`。

---

## 3. 修复方案

**文件**：`CommonProjectService.java`

| 逻辑 | 说明 |
|------|------|
| 检测 org 变更 | 对比 DB 中现有 `organizationId` |
| 校验新组织 | 组织存在性检查 |
| 角色迁移 | `updateProjectUserRoleOrganizationId()` |
| 管理员补全 | 新组织下 `checkOrgRoleExit()` |
| selective update | **仅当 org 未变更**时 `setOrganizationId(null)`，避免覆盖 intentional 更新 |

**前端**：`addProjectModal.vue` — 编辑态不再 `:disabled="!isXpack"`；仅**创建**时默认当前组织。

---

## 4. 变更文件

| 文件 | 说明 |
|------|------|
| `CommonProjectService.java` | update 逻辑 |
| `addProjectModal.vue` | 编辑时组织可选 |

---

## 5. 验证

1. 编辑示例项目，修改所属组织 → 保存  
2. 列表与详情显示新组织  
3. 项目成员 `user_role_relation.organization_id` 与新组织一致（抽样 DB）
