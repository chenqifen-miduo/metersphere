# BUG-SYS-002：开发环境未认证 API 返回 500 而非 401

> **日期**：2026-07-04  
> **关联任务**：全系统测试 / 全局安全  
> **影响范围**：系统、组织、项目管理未带 Token 的 API 调用  
> **严重程度**：P2

---

## 1. 问题现象

不带 `X-AUTH-TOKEN` / `CSRF-TOKEN` 调用例如：

- `POST /organization/member/list`
- `POST /project/environment/list`

**期望**：HTTP 401/403  
**实际**：HTTP 500，`No static resource index.html`

---

## 2. 根因分析

Shiro `authc` 未认证时 forward 到 `/` → `IndexController` 返回 `index.html`。  
本地 dev 未构建 `frontend/dist`，静态资源不存在 → 500。

---

## 3. 修复方案

| 文件 | 说明 |
|------|------|
| `MsAuthenticationFilter.java` | 新增：API/JSON 请求 `onAccessDenied` 返回 401 + `ResultHolder` |
| `ShiroConfig.java` | `getFilters().put("authc", new MsAuthenticationFilter())` |

`isApiRequest` 判定：非 `/`、`/login`；或 Accept/Content-Type 含 `application/json`；或非静态资源扩展名。

---

## 4. 验证

```bash
curl -X POST http://localhost:8081/organization/member/list -H "Content-Type: application/json" -d "{}"
# 期望 HTTP 401，body 含 code 100401
```

---

## 5. 备注

生产环境有 `frontend/dist` 时原问题可能不明显；本修复统一 API 语义。
