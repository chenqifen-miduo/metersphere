# BUG-PLAN-001：已完成计划「执行历史」Tab 为空

> **日期**：2026-07-04  
> **关联任务**：全系统测试 / 测试计划模块  
> **影响范围**：测试计划详情 → 执行历史 Tab  
> **严重程度**：P1

---

## 1. 问题现象

打开已完成计划（如 `v1.0版本发布计划`，15/15）→「执行历史」Tab 显示「暂无数据」。

**关联 API**：`POST /test-plan/his/page`

---

## 2. 根因分析

1. **`listHis` SQL 仅查 `exec_task`**：演示数据可能仅有用例级执行记录，无计划级 `exec_task`（`resource_id = testPlanId`）。  
2. **缺少 `deleted = 0` 过滤**：迁移 3.5.0 后 `exec_task.deleted` 可能过滤掉有效历史。

手动「生成报告」产生的 `test_plan_report` 未纳入执行历史展示。

---

## 3. 修复方案

**文件**：`backend/services/test-plan/.../ExtTestPlanMapper.xml` — `listHis`

- `exec_task` 增加 `et.deleted = 0`
- `UNION ALL` 已完成报告（`test_plan_report`，`result_status in ('SUCCESS','ERROR','FAKE_ERROR')`）
- 排除已通过 `api_report_relate_task` 关联到 exec_task 的报告，避免重复

---

## 4. 变更文件

| 文件 | 说明 |
|------|------|
| `ExtTestPlanMapper.xml` | `listHis` 查询扩展 |

---

## 5. 验证步骤

1. 打开已完成测试计划 → 执行历史 Tab  
2. 若存在手动/自动报告或计划级 exec_task，应出现记录行  
3. 可选：`POST /test-plan/his/page`，body 含 `testPlanId`
