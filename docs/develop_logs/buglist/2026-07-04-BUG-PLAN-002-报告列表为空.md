# BUG-PLAN-002：生成报告后报告列表仍为空

> **日期**：2026-07-04  
> **关联任务**：全系统测试 / 测试计划模块  
> **影响范围**：测试计划 → 生成报告 → 报告 Tab / `#/test-plan/testPlanReport`  
> **严重程度**：P1

---

## 1. 问题现象

已完成计划点击「生成报告」，前端提示成功，但报告列表「暂无数据」。  
详情页可能可打开，列表 SQL 查不到记录。

**关联 API**：`POST /test-plan/report/auto-gen`、`POST /test-plan/report/page`

---

## 2. 根因分析

1. **`postHandleReport` NPE**：`passThreshold` 为 null 时 `passRate >= passThreshold` 抛异常，被 `genReport` catch 吞掉。  
2. **报告停留在 `result_status = '-'`**：列表 SQL 条件 `result_status in ('SUCCESS','ERROR','FAKE_ERROR')` 过滤掉未完成后置处理的报告。  
3. **无 `test_plan_config` 时**：`preGenReport` 将 `passThreshold` 设为 null，触发上述问题。

---

## 3. 修复方案

**文件**：`TestPlanReportService.java`

| 改动 | 说明 |
|------|------|
| `preGenReport` | `passThreshold` 默认 `100.0`（与 `TestPlanManagementService` 一致） |
| `postHandleReport` | `executeResult` / `passRate` / `passThreshold` 空值保护；阈值比较使用局部变量 |

---

## 4. 变更文件

| 文件 | 说明 |
|------|------|
| `TestPlanReportService.java` | 预生成与后置处理 |

---

## 5. 验证步骤

1. 打开已完成计划 → 生成报告  
2. 进入报告 Tab，列表应含新报告（含通过率、结果状态）  
3. DB：`test_plan_report.result_status` 应为 `SUCCESS` 或 `ERROR`
