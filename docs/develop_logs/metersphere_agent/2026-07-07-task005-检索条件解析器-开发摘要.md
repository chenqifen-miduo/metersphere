## 3. 解析流程

1. `filters.moduleIds` 已有 → 直接使用  
2. `query` → 模块树匹配 → 展开子孙 moduleIds  
3. 未限定范围 → `query` 作为 keyword  
4. 叠加 `priority` / `lastExecuteResult` / `tags` filters  

模块未命中时 warning：`MODULE_NOT_MATCHED_KEYWORD_FALLBACK`
