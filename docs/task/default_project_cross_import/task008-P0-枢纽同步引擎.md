## 1. 任务目标

实现业务项目 →【米多公司默认项目】的用例与计划「测试计划」Tab 同步：事件尽力同步 + 每日 0 点对账 + 手动同步。导入副本**不**登记为同步源（不回写）。

---

## 2. 任务清单

### 2.1 数据模型

- [ ] `default_hub_case_map`：`biz_project_id`, `biz_case_id`, `hub_case_id`, `content_hash`, `updated_at`  
- [ ] `default_hub_plan_map`：同上结构（计划）  
- [ ] `default_hub_sync_job`：类型（EVENT/CRON/MANUAL）、范围、状态、进度、错误摘要  
- [ ] Flyway 迁移 + 索引（按 biz 唯一）  

### 2.2 同步内容

- [ ] **用例**：基本信息 + 前置/备注/步骤；枢纽侧未评审、未执行；无附件/关联  
- [ ] **计划**：`test_plan` 约定字段 + `test_plan_document`（含图片文件拷贝与引用重写）；无用例关联/组/定时/报告  
- [ ] 落点：用例落入 `ref_project_id` 对应文件夹下（路径策略文档化）  
- [ ] 删除：业务删用例/计划 → 枢纽镜像删除  

### 2.3 触发

- [ ] 写路径 Outbox/事件：创建、更新、删除（用例；计划文档保存）  
- [ ] Cron：每日 **0:00**（时区约定，建议 Asia/Shanghai）+ 多节点抢锁（ShedLock 等）  
- [ ] 手动：`POST .../default-hub/sync`（`projectId` 可选）；UI 按钮 + 上次同步时间  
- [ ] 失败重试队列；任务进度可查  

### 2.4 防环

- [ ] 仅已建立 hub map 的业务对象参与同步  
- [ ] 从枢纽导入的副本**不**写入 map（仅可选审计 `imported_from_hub_*`）  
- [ ] T16：改导入副本，枢纽源不变

## 3. 验收标准

- [ ] T12：业务新建用例/计划 → 枢纽出现镜像（计划仅 Tab）  
- [ ] T13：业务修改 → 枢纽更新  
- [ ] T14：定时 + 手动任务成功，进度可见  
- [ ] T16：导入副本不回写  
- [ ] 灰度开关：可关闭事件/定时而不影响主业务写入
