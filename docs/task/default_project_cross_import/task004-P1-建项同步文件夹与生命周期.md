# task004 - P1 建项同步文件夹与生命周期

> **阶段**：S3  
> **预估工期**：2.5–3 天  
> **前置依赖**：task002、task003  
> **阻塞任务**：task008  
> **关联方案**：§3.4、§3.9

---

## 1. 任务目标

创建业务项目时：默认项目根下建同名 `FOLDER`（`ref_project_id`）+#9 本项目根下同名文件夹；并处理改名/删除/禁用副作用。

---

## 2. 任务清单

- [ ] 建项成功后（主事务外或可降级）：  
  - 默认项目：`FOLDER` + `ref_project_id=新项目Id`  
  - 本项目：`FOLDER` 同名  
- [ ] 重名加后缀；幂等（已存在同 `ref_project_id` 则跳过）  
- [ ] 改名：仅按 `ref_project_id` 更新默认项目文件夹名；禁止纯名称匹配  
- [ ] 删除/软删业务项目：默认项目对应文件夹**随删**（级联）  
- [ ] 禁用业务项目：对应文件夹树中**隐藏**；导入读接口仍可用  
- [ ] 失败不阻断建项；日志 + 可选补偿接口  
- [ ] 单测/集成：建项双侧文件夹、改名、删除、禁用  

---

## 3. 验收标准

- [ ] T6 / T6b / T6c 全部通过  
- [ ] 人工改过文件夹显示名后再改项目名：仍按 `ref_project_id` 跟随项目名（方案默认）
