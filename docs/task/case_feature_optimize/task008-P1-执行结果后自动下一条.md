# task008 - P1 执行结果后自动下一条

> **阶段**：P1  
> **预估工期**：0.5–1 人日  
> **前置依赖**：[task005](task005-P0-详情上下条导航状态修复.md)  
> **阻塞任务**：无  
> **关联总览**：[task000](task000-实施总览与依赖关系.md)  
> **任务状态**：待开始

---

## 1. 任务目标

在功能用例详情中，用户点击用例级结果按钮（**通过 / 失败 / 阻塞 / 跳过**）并保存成功后，**自动进入下一条**用例。

---

## 2. 触发范围

| 触发 | 不触发 |
|------|--------|
| `handleSetCaseResult`（`addStep.vue` 底部四按钮） | 步骤级执行结果变更 |
| 保存成功（`persistCase`）后 | 内容编辑自动保存 |
| | 批量修改执行结果 |

---

## 3. 任务清单

### 3.1 自动跳转逻辑

- [ ] `tabDetail.vue` 的 `handleSetCaseResult`：在 `persistCase(true)` 成功后  
  - 若 `canGoNext === true` → `emit('nextCase')`  
  - 若已是最后一条 → `Message.info` 提示（i18n 新增 `lastCaseTip`）  
- [ ] `caseDetailDrawer.goNextCase()` 调用 `detailDrawerRef.openNextDetail()`  

### 3.2 可选增强（非本期必做）

- [ ] 用户偏好开关「提交后自动下一条」（参考测试计划 `autoNextTip`）

---

## 4. 涉及文件

| 文件 | 改动 |
|------|------|
| `tabContent/tabDetail.vue` | 保存成功后跳转 |
| `caseDetailDrawer.vue` | 确保 nextCase 链路通畅 |
| `locale/zh-CN.ts`、`locale/en-US.ts` | 最后一条提示文案 |

---

## 5. 测试用例

| 场景 | 预期 |
|------|------|
| 非最后一条点「通过」 | 保存成功并进入下一条 |
| 非最后一条点「失败/阻塞/跳过」 | 同上 |
| 最后一条点结果按钮 | 提示已是最后一条，停留当前 |
| 改步骤结果 | 不自动跳转 |
| 自动跳转后上一条 | 可返回刚操作的用例 |

---

## 6. 验收标准

- [ ] 用例级结果四按钮保存成功后自动下一条  
- [ ] 最后一条有明确提示  
- [ ] 步骤级结果不触发自动跳转
