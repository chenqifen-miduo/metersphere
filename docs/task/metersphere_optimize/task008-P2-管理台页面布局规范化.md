# task008 - P2 管理台页面布局规范化

> **阶段**：P2  
> **预计工期**：2 天  
> **前置依赖**：[task005](task005-P1-UI样式覆盖治理与设计token收敛.md)  
> **阻塞任务**：无  

---

## 1. 任务目标

统一工作台、设置、用例、接口、测试计划等管理台页面的信息架构和操作区，让用户在不同业务模块之间切换时保持一致的扫描和操作习惯。

---

## 2. 页面模板

建议沉淀以下页面模板：

| 模板 | 用途 |
|------|------|
| 左树右表 | 用例、接口、场景、文件管理 |
| 顶部筛选 + 表格 | 缺陷、任务中心、报表列表 |
| 详情抽屉 | 用例详情、接口详情、缺陷详情 |
| 创建/编辑弹窗 | 轻量配置、新增分组、新增标签 |
| 创建/编辑抽屉 | 表单复杂、字段较多的业务对象 |
| 工作台卡片 | 指标、待办、趋势图 |

---

## 3. 任务清单

### 3.1 定义页面结构规范

统一：

- 页面标题区。
- 面包屑区域。
- 筛选区域。
- 主操作按钮位置。
- 批量操作栏位置。
- 表格分页位置。
- 详情抽屉宽度。
- 空状态和错误状态。

### 3.2 选择试点页面

建议试点：

- 用例管理列表。
- 接口管理列表。
- 测试计划列表。

### 3.3 抽象复用组件

可沉淀：

- `MsPageShell`
- `MsListToolbar`
- `MsFilterPanel`
- `MsTreeTableLayout`
- `MsDetailDrawerLayout`

### 3.4 窄屏策略

明确：

- 最小可用宽度。
- 表格横向滚动。
- 关键列固定。
- 操作列固定。
- 筛选项折叠规则。

---

## 4. 验收标准

- [ ] 试点页面交互和布局一致。
- [ ] 批量操作、筛选、分页、详情查看位置稳定。
- [ ] 空状态和加载状态统一。
- [ ] 后续页面有可复制模板。
- [ ] 窄屏下关键操作不遮挡、不溢出。
