## 1. 任务目标

减少散落的 `console.log/error`，统一用户反馈和开发诊断方式，让接口错误、业务操作失败、生产异常有一致处理策略。

---

## 2. 涉及范围

| 范围 | 说明 |
|------|------|
| `frontend/src/api/http/*` | Axios 封装、状态码处理、取消请求 |
| `frontend/src/utils/monitor.ts` | 生产异常上报 |
| `frontend/src/store` | store 中的异常处理 |
| 登录、设置、表格相关页面 | 高频用户操作失败提示 |

---

## 3. 任务清单

### 3.1 梳理 console 来源

统计并分类当前 `console.*`：

- 请求失败。
- 用户操作失败。
- 开发调试。
- 第三方库兼容。
- 可删除的历史日志。

### 3.2 设计统一错误策略

| 错误类型 | 处理方式 |
|----------|----------|
| API 业务错误 | 请求层统一解析，必要时 Message 提示 |
| 401 / 登录失效 | 统一跳转登录或弹出登录态失效提示 |
| 403 / 无权限 | 跳转无权限页或当前操作提示 |
| 用户操作失败 | 业务页面 Message/Notification |
| 动态 import 失败 | 保留系统升级提示 |
| 开发诊断日志 | 仅开发环境输出 |
| 生产异常 | 上报或静默降级 |

### 3.3 封装统一 logger

建议新增：

`frontend/src/utils/logger.ts`

能力：

- `logger.debug`
- `logger.info`
- `logger.warn`
- `logger.error`
- 根据环境控制输出。
- 支持后续接入异常上报。

### 3.4 替换高频裸 console

优先治理：

- `frontend/src/api/http/*`
- `frontend/src/store/modules/*`
- `frontend/src/App.vue`
- 登录相关页面。
- 设置相关页面。
- `ms-table` 相关逻辑。

- [ ] 生产环境不输出无意义 `console.log`。
