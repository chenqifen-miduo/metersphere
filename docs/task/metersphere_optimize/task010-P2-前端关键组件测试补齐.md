## 1. 任务目标

给高风险基础设施组件补最小测试保护，确保后续拆分 `ms-table`、请求层、权限路由、重功能懒加载时有回归保障。

---

## 2. 测试范围

| 范围 | 测试重点 |
|------|----------|
| `ms-table` | 选择、分页、筛选、列配置、批量操作 |
| 请求拦截器 | token、401、403、错误提示、取消请求 |
| 权限路由 | 登录态、白名单、无权限跳转 |
| 文件上传 | 成功、失败、大小限制、队列 |
| PDF 导出 | 懒加载、导出入口、失败提示 |
| 富文本 | 基础渲染、输入、附件或图片 |
| 脑图编辑器 | 初始化、选择节点、保存 |

---

## 3. 任务清单

### 3.1 建立测试优先级

优先级：

1. `ms-table`
2. 请求层
3. 权限路由
4. 文件上传
5. PDF / 富文本 / 脑图 smoke test

### 3.2 补齐 ms-table 测试

覆盖：

- 单选。
- 多选。
- 全选。
- 禁用选择。
- 分页切换。
- 筛选确认。
- 列设置。
- 批量操作触发。

### 3.3 补齐请求层测试

覆盖：

- token 注入。
- 401 行为。
- 403 行为。
- 网络错误。
- 业务错误。
- 重复请求取消。

### 3.4 补齐路由权限测试

覆盖：

- 未登录访问白名单。
- 未登录访问受保护页面。
- 已登录无权限访问。
- 动态路由加载。

### 3.5 重功能 smoke test

覆盖：

- PDF 导出入口可打开。
- 富文本编辑器可渲染。
- 脑图编辑器可初始化。
- 代码编辑器懒加载后可渲染。
