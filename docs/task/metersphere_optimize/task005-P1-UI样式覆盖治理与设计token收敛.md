# task005 - P1 UI 样式覆盖治理与设计 token 收敛

> **阶段**：P1  
> **预计工期**：2 天  
> **前置依赖**：[task001](task001-P0-前端工程基线清理.md)  
> **阻塞任务**：[task008](task008-P2-管理台页面布局规范化.md)  

---

## 1. 任务目标

减少 `:deep()` 和 `!important` 的扩散，沉淀统一 UI token、组件变体和页面样式规范，降低 Arco/Element 组件升级和主题适配成本。

---

## 2. 当前问题

静态扫描显示：

- `:deep()` 使用较多。
- `!important` 使用较多。
- 多个页面重复覆盖表格、菜单、弹窗、抽屉、筛选器样式。
- 页面级样式与组件级样式边界不清。

这些问题会导致：

- 主题切换风险变高。
- 样式优先级难维护。
- 新页面复制旧页面样式补丁。
- 第三方组件升级成本增加。

---

## 3. 任务清单

### 3.1 样式覆盖分类

对 `:deep()` 和 `!important` 分类：

| 类型 | 处理策略 |
|------|----------|
| 第三方组件必要覆盖 | 收敛到统一组件或全局主题覆盖 |
| 页面临时补丁 | 评估移除或沉淀为组件变体 |
| 重复表格样式 | 收敛到 `ms-table` |
| 重复弹窗/抽屉样式 | 收敛到 `ms-dialog`、`ms-drawer` |
| 重复筛选器样式 | 收敛到筛选区组件 |

### 3.2 建立设计 token

建议覆盖：

- 间距。
- 圆角。
- 边框。
- 字号。
- 行高。
- 表格行高。
- 抽屉宽度。
- 筛选区高度。
- 页面背景。
- 卡片/面板边界。

### 3.3 治理高频组件

优先治理：

- `ms-table`
- `ms-dialog`
- `ms-drawer`
- `ms-menu`
- `ms-advance-filter`
- `ms-empty`
- `ms-pagination`

### 3.4 输出 UI 规范

建议新增：

`docs/task/metersphere_optimize/ui-style-guideline.md`

内容包括：

- 页面级样式允许范围。
- 组件级样式收敛原则。
- 何时允许 `:deep()`。
- 何时允许 `!important`。
- 新增业务页面布局建议。

---

- [ ] 输出 UI 样式治理清单。
- [ ] 建立常用设计 token。
- [ ] 高频组件减少重复样式覆盖。
- [ ] 暗色主题和主题色切换不受影响。
