## 3. 页面结构

### 3.1 列表页

列：报告名称、关联范围（计划/时间）、生成人、更新时间；操作：**详情 | 编辑 | 删除**（顺序与 task001 一致时可复用习惯）。

工具栏：**一键生成报告**。

### 3.2 一键生成弹窗

| 字段 | 说明 |
|------|------|
| 报告名称 | 默认「测试报告-{yyyyMMdd}」可改 |
| 测试计划 | 可选；空=项目范围（与后端约定一致） |
| 时间范围 | 可选（若 API 支持） |

### 3.3 详情/编辑页章节

| 章节 | 可编辑 | 说明 |
|------|--------|------|
| 一、版本概览 | ✅ 文字 | 生成带默认值 |
| 二、测试范围与目的 | ✅ 仅「测试内容」 | **无测试依据** |
| 三、质量评估 3.1 | 只读表 + 刷新统计 | 旁注通过率公式 |
| 三、3.2 缺陷图 | 只读图 + 刷新 | 图1 处理人×状态；图2 类型 |
| 四、遗留问题与风险 | 自动列表 + 可补说明 | 阻塞+失败 |
| 五、测试结论与建议 | ✅ | 可有默认草稿 |
| 六、附件 | ❌ | 不渲染 |
| 页脚 | ✅ | 撰写人、日期默认当前用户/当天 |
