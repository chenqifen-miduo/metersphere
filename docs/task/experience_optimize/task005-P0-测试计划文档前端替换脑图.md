## 3. 模板章节（对齐方案 §3.2.2）

首次进入且无文档时，注入下列章节骨架（可用 HTML 表格占位）：

1. 文档信息（编号、版本、所属项目、测试模块、编制日期、编制人、审核人）  
2. 项目背景  
3. 测试目标  
4. 测试范围（范围内 / 范围外）  
5. 测试策略  
6. 测试重点（主流程 / 规则与边界 / 权限与安全）  
7. 测试环境与数据  
8. 准入、暂停与退出标准  
9. 测试进度  
10. 人员与职责  
11. 缺陷管理  
12. 风险与应对  
13. 测试交付物  
14. 审批记录  

**自动填充**：所属项目、测试模块/标题建议、编制日期、编制人、文档编号（来自 task004 meta 或前端拼接）。

样例参考：方案附录「商城-订单改址-测试计划」；若仓库无该 md，以方案章节为准内置常量模板。

## 4. 任务清单

### 4.1 Tab 与路由

1. Tab 文案改为「测试计划」（中英繁）。  
2. 替换 `detail/plan/index.vue` 实现：加载 GET document → 空则套模板 → 编辑 → POST 保存。  

### 4.2 工具栏

| 按钮 | 行为 |
|------|------|
| 保存 | POST；成功 Message |
| 预览 | 只读态或弹层预览（建议做） |
| 重置为模板 | Modal 二次确认后覆盖本地编辑器内容（需再点保存才落库，或确认后直接保存——实现选一种并写清） |

### 4.3 权限

- 查看：`PROJECT_TEST_PLAN:READ`  
- 编辑/保存：`PROJECT_TEST_PLAN:READ+UPDATE`  
- 无编辑权限：只读展示  

### 4.4 移除脑图入口

- 前端不再渲染 minder、不调用 minder 保存。  
- 发版说明：原「测试规划」脑图不再提供；用例关联请用计划内其它 Tab。  

### 4.5 API 封装

- `frontend/src/api/modules/test-plan/document.ts`（路径按项目惯例）  
- 类型：`models/testPlan/document.ts`
