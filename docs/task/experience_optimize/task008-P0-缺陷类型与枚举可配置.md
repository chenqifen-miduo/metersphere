# task008 - P0 缺陷类型字段与枚举可配置

> **阶段**：P0  
> **预估工期**：3–5 人日  
> **前置依赖**：无（宜在 task007 图2 联调前完成）  
> **阻塞任务**：task006 图2 完整聚合、task007 饼图展示  
> **关联方案**：[体验优化产品方案](../../summary/MeterSphere-体验优化-产品方案-2026-07-17.md) §3.4.6  
> **任务状态**：待开始  
> **标注**：涉及缺陷工作流与枚举变更，实现后须**人工审核**

---

## 1. 任务目标

1. 缺陷管理新增 **【缺陷类型】** 字段，选项由管理员自由增删改。  
2. 原有 **状态、严重程度** 亦改为管理员可自由添加、修改、删除（删除已占用选项须禁用或二次确认，避免破坏历史数据）。  
3. 测试报告图1 处理人统一取 **`handleUser`**；图2 按「缺陷类型」聚合。

---

## 2. 范围说明

| 项 | 要求 |
|----|------|
| 缺陷类型 | 新建系统字段（或模板自定义字段落地为固定业务字段）；创建/编辑缺陷可选；列表可展示/筛选 |
| 状态 | 管理员维护选项列表（增/改/删或禁用） |
| 严重程度 | 同上 |
| 配置入口 | 对齐现有「项目设置 / 缺陷模板」或组织级模板机制 |
| 权限 | 仅管理员（或具备模板配置权限的角色）可改选项 |

---

## 3. 任务清单

### 3.1 后端

1. 缺陷类型字段 DDL / 模板字段初始化（Flyway，SQL 须人工审）。  
2. 状态、严重程度选项的 CRUD API（若已有模板字段 API 则扩展，避免重复造轮子）。  
3. 删除策略：选项若被缺陷引用 → **禁用** 或二次确认后才允许删；禁止静默物理删导致脏数据。  
4. 查询/统计接口支持按缺陷类型过滤与分组（供 task006 图2）。  

### 3.2 前端

1. 缺陷创建/编辑表单增加「缺陷类型」。  
2. 缺陷列表增加类型列与筛选（可选，建议本期做）。  
3. 管理员配置页：维护缺陷类型 / 状态 / 严重程度选项。  
4. i18n：`bugManagement.bugType` 等。  

### 3.3 与报告联动

- task006 图2 聚合字段 = 本任务落地的缺陷类型字段 key。  
- 无类型值 →「未分类」。  
- 图1 处理人 = `handleUser`（不取创建人）。  

---

## 4. 测试用例

| 场景 | 预期 |
|------|------|
| 管理员新增缺陷类型选项 | 创建缺陷时可选到新选项 |
| 管理员修改/禁用选项 | 新单不可选禁用项；历史数据仍可展示原值 |
| 非管理员 | 不可改选项配置 |
| 报告生成 | 图2 按类型计数正确；图1 按 handleUser×状态 |

---

## 5. 验收标准

- [ ] 缺陷表单可见「缺陷类型」并可保存。  
- [ ] 管理员可对缺陷类型、状态、严重程度选项增删改（删除有保护）。  
- [ ] 报告聚合可按缺陷类型、`handleUser` 取数。  

---

## 6. 风险

- 状态自由配置可能影响现有缺陷工作流、看板与统计口径，需与产品确认默认内置状态集。  
- 严重程度变更影响优先级展示与过滤，须发版说明。  
- 本任务属**敏感业务逻辑**，合并前人工审核默认值与迁移脚本。
